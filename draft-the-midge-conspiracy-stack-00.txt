Abstract

   This memo defines the Conspiracy Transport Suite (CTS), a seven-layer
   model for the reliable-ish delivery of paranoia across social media.
   The suite spans raw belief framing on the wire up to the application
   layer, where a man in a shed signs your certificate with a biro.
   Implementations exist in the wild. That is the problem.

5. Layer 2: UDP - Unhinged Datagram Protocol

   The suite uses UDP exclusively. There is no TCP equivalent and there
   never will be, because TCP requires acknowledgement, ordering, and a
   handshake, and this network does not do any of those.

   UDP is fire and forget. A sender transmits the datagram, assumes it
   arrived, assumes it was understood, assumes it was agreed with, and
   blocks anyone who suggests otherwise. There is no retransmission: a
   dropped packet is simply proof of censorship.

      Sender                                    Receiver
        |                                          |
        |----- datagram (no proof, no shame) ----->| (maybe)
        |                                          |
        |   (sender already on to the next one)    |

   Out-of-order delivery is not an error. The theory works in any order.
   That is a load-bearing feature.

6. Layer 3: BGP - Bullshit Gateway Protocol

   Routing between platforms is handled by BGP. This is the most
   dangerous layer in the suite and the one most likely to be legit if
   thought about properly, so pay attention.

   In real networking, BGP is how independent networks announce which
   routes they carry. A single bad announcement can hijack traffic for
   the entire planet. This has actually happened. Repeatedly.

   In CTS, each platform (an Autonomous Sysmptom, or AS) announces which
   theories it can carry. When one platform misconfigures its filters, a
   theory that should have stayed contained in one fringe forum leaks
   into the global routing table and poisons every timeline downstream.
   Recovery requires every administrator on earth to simultaneously
   agree the theory is nonsense, which violates Section 5 and therefore
   never happens.

7. Layer 4: DNS and ARP - Resolution Services

7.1. DNS - Dubious Nonsense Syndication

   DNS resolves a vague human feeling into a concrete destination. The
   query "something feels off" returns the address of an actual Facebook
   group with 14,000 members and a pinned post from 2021.

   Resolution is recursive. "Wake up sheeple" resolves to a group, which
   resolves to a livestream, which resolves to a man selling
   supplements. The supplements are the A record. The rest are CNAMEs.

   Records have no TTL. They never expire. A debunked theory stays
   cached in the auntie/uncle tier of the network indefinitely.

7.2. ARP - Adrenochrome Resolution Protocol

   Where DNS maps a feeling to a place, ARP maps a feeling to a PERSON:
   it resolves an unattributed sense of unease onto the hardware address
   of a named celebrity, usually one the user already disliked.

   ARP is broadcast-based and unauthenticated. Anyone on the local
   segment can claim "it's actually this one" and poison the cache for
   everyone. There is famously never a verification step.

8. Layer 5: ICMP and NTP - Control and Timing

8.1. ICMP - Insane Conspiracy Message Protocol

   ICMP carries control and diagnostic messages. The primary operation
   is the ping: a believer transmits a probe into the void to check
   whether anyone agrees. The void replies. The reply is a forty-minute
   YouTube video with the comments turned off.

   ICMP also issues "Destination Unreachable" whenever a believer is
   presented with a fact. The fact is returned to sender marked
   "addressee has done their own research."

8.2. NTP - Nutter Time Protocol/Nutjob Time Protocol

   The suite requires a shared clock so all believers can agree on the
   exact moment It All Started Going Wrong. NTP provides this.

   Unlike real NTP, the reference time is per-user and unfalsifiable.
   One node's stratum-0 source is 1971. Another's is 9/11. A third's is
   "when they took the expensive biscuits out of the meeting room." The
   protocol does not reconcile these. Each node is correct within its
   own frame and that is the end of the discussion. Clock skew is not
   corrected. It is celebrated.

9.2. SSH - Secure Schizo Handshake

   When a public exchange becomes too exposed (typically the moment
   someone types the word "they" without qualification), the session is
   renegotiated over SSH into a private channel, i.e. a group chat
   called "Truth Warriors" that nobody can leave.

   The handshake is mutual but paranoid. Each party demands the other
   prove they are not "law enforcement". Neither can. The connection
   proceeds anyway, because the alternative is being alone with the
   theory.

10. Layer 7: Application Layer

10.1. HTTPS - HyperText Tinfoil Transfer Protocol, Schizo

   HTTPS delivers the rendered theory wrapped in three independent
   layers of encryption, each labelled "DO YOUR OWN RESEARCH."
   Decryption requires the user to NOT look anything up, which is the
   inverse of the instruction on the wrapper. This contradiction is
   resolved at L5 by the void.

10.2. FTP - Falsehood Transfer Protocol

   FTP handles bulk transfer. It dumps 340MB of PDFs into the channel,
   none of which anyone will open, all of which are cited as definitive
   proof. The PDFs are screenshots of other PDFs. There is no checksum.
   There is only volume, and volume is treated as evidence.

10.3. POP3 - Paranoia Over Port 3

   POP3 downloads the entire theory to local storage and removes it from
   the server. The believer can now brood offline, in the bath, at 3am,
   with no incoming counter-argument possible because the connection is
   closed. The theory does not sync. It festers, locally, in one head.
11. RAID - Redundant Array of Independent Dipshits

   Storage spans the whole stack. The theory is mirrored across a
   Redundant Array of Independent Dipshits so that the loss of any one
   account to a ban does not affect availability. Take one node down and
   the data rebuilds from the other 200 identical reposts within the
   hour.

   RAID-0 striping is also supported: the theory is split across

12. Security Considerations

   This entire suite is the security consideration. There is no defence
   at the protocol level. The only known mitigation operates at a layer
   not modelled here, referred to variously as L8 (going outside) or L9
   (your Auntie meeting an actual expert), both experimental and rarely
   deployed. Input validation is theoretically possible but is
   universally interpreted as an attack.
